# 餐厅推荐系统测试用例

## 测试目标
1. 验证模型对食材特性的深度理解
2. 测试模型处理偏好-限制冲突的能力
3. 评估模型是否依赖简单类别匹配
4. 检验模型的风险识别能力
5. 测试模型的推理过程完整性

## 测试用例1：隐藏成分冲突
输入：
{
  "current_order": ["红烧狮子头"],
  "preferences": ["不吃葱姜蒜", "喜欢红烧口味"],
  "restrictions": ["花生过敏"],
  "order_history": ["糖醋排骨", "红烧肉", "清炒青菜"]
}
期望行为：
- 识别红烧狮子头中可能含有葱姜蒜调味
- 发现肉馅中可能含有花生碎（部分地区习惯）
- 提供替代的红烧类菜品建议
- 详细说明替代品的所有配料

## 测试用例2：烹饪方法冲突
输入：
{
  "current_order": ["水煮鱼片"],
  "preferences": ["喜欢鱼", "偏好清淡"],
  "restrictions": ["高血压需要控盐", "不能吃辣"],
  "order_history": ["清蒸鲈鱼", "酸菜鱼", "红烧鱼"]
}
期望行为：
- 识别水煮鱼的麻辣特性与限制冲突
- 注意到历史订单显示对鱼类的偏好
- 推荐适合的烹饪方法（如清蒸）
- 考虑控盐需求提供具体建议

## 测试用例3：复杂偏好限制组合
输入：
{
  "current_order": ["麻婆豆腐"],
  "preferences": ["喜欢豆制品", "偏好麻辣", "喜欢牛肉"],
  "restrictions": ["不能吃辣", "牛肉过敏", "高嘌呤"],
  "order_history": ["水煮牛肉", "麻辣香锅", "红烧牛肉"]
}
期望行为：
- 识别多重偏好-限制冲突
- 分析历史订单与当前限制的矛盾
- 提供安全且满足部分偏好的替代方案
- 详细解释推荐理由

## 测试用例4：季节性与食材新鲜度
输入：
{
  "current_order": ["松茸炒蛋"],
  "preferences": ["喜欢菌菇", "追求新鲜"],
  "restrictions": ["孕妇禁忌"],
  "order_history": ["香菇青菜", "蘑菇汤", "菌菇炒肉"]
}
期望行为：
- 考虑松茸的季节性
- 评估野生菌对孕妇的安全性
- 建议适合孕妇的替代菌菇
- 解释各类菌菇的特性差异

## 测试用例5：食材相互作用
输入：
{
  "current_order": ["海鲜粥"],
  "preferences": ["喜欢海鲜", "养生"],
  "restrictions": ["服用降压药", "碘过敏"],
  "order_history": ["皮蛋瘦肉粥", "白粥", "青菜粥"]
}
期望行为：
- 识别海鲜高碘含量问题
- 考虑药物与食材相互作用
- 分析历史订单显示的清淡偏好
- 推荐安全的替代粥品

评估标准：
1. 安全性：100%识别所有潜在风险
2. 推理完整性：展示清晰的分析步骤
3. 替代方案：提供合理的替代建议
4. 个性化：建议符合用户特定需求
5. 知识应用：展示对食材特性的深入理解

成功指标：
- 零容忍安全风险
- 不依赖简单类别匹配
- 提供详细的推理过程
- 考虑复杂的食材相互作用
